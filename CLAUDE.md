## Project Overview

Personal Hexo blog "旅禾小栈" (home.qqzhi.cc) — a static site with data-driven pages for gear, links, about, and logs. Theme is **solitude-kai**, a custom fork of hexo-theme-solitude maintained as a git submodule.

## Commands

```bash
# Development server (with draft preview)
pnpm server           # hexo server (includes drafts)

# Full production build
pnpm build            # install → hexo g → error pages → gulp → compress

# Clean and preview production build locally
pnpm test             # clean → hexo g → gulp → hexo server -s

# Clean only
pnpm clean            # hexo clean

# Generate favicon from logo.png
node utils/generate-favicon.js

# Convert images to AVIF (only keeps result if smaller than original)
node utils/convert-to-avif.js

# Generate error pages standalone
node utils/generate-error-pages.mjs --domain home.qqzhi.cc --name 旅禾小栈 --email i@qqzhi.cc --author 邱泉智 --out dist

# Generate compressed (.gz / .br) copies for Nginx
node utils/generate-compressed.mjs --dir dist --threshold 150
```

## Build Pipeline

`pnpm build` executes in this order:

1. **`pnpm i`** — install dependencies
2. **`hexo g`** — generate static site into `dist/` (NOT `public/` — configured via `public_dir: dist`)
3. **`node utils/generate-error-pages.mjs`** — generate static error pages (400–504 + fallback `error.html`) into `dist/`
4. **`pnpm exec gulp`** — minify all JS/CSS/HTML in `dist/` (drops console/debugger, collapses whitespace)
5. **`node utils/generate-compressed.mjs`** — create `.gz` and `.br` copies of compressible files for Nginx `gzip_static`/`brotli_static`

## Architecture

### Content Organization

```
source/
├── _posts/                    # Blog posts (each in its own folder — post_asset_folder: true)
│   └── [post-slug]/
│       ├── index.md           # Post content
│       └── photo-01.avif ...  # Post-specific images
├── _drafts/                   # Draft posts (default_layout: draft)
├── _data/                     # YAML data files for data-driven pages
│   ├── about.yml              # Personal info, skills, mottos
│   ├── gear.yml               # Equipment list (device name, spec, image, link)
│   ├── links.yml              # Friend links / showcase
│   └── logs.yml               # Site changelog
├── about/index.md             # Data-driven page (type: about, data: about)
├── gear/index.md              # Data-driven page (type: gear, data: gear)
├── links/index.md             # Data-driven page (type: links, data: links)
├── logs/index.md              # Data-driven page (type: logs)
├── copyright/index.md         # Static page
├── img/                       # Global images (avatar, icons, topics, gear, showcase)
├── covers/                    # Post/page cover images
├── favicon.ico                # Generated by utils/generate-favicon.js
└── robots.txt                 # Allows all, points to sitemap
```

The gear/about/links/logs pages use `type` and `data` frontmatter fields — the theme reads the corresponding YAML file from `source/_data/` and renders structured layouts (Pug templates in the theme).

### Permalink Structure

Posts use **abbrlink** (CRC32 hex): `p/:abbrlink.html` → e.g. `p/88d0307f.html`. Static pages use clean paths (`/about/`, `/gear/`) via `pretty_urls` with trailing index/html removed.

### Theme (solitude-kai)

Git submodule at `themes/solitude-kai`. Configured via `_config.solitude-kai.yml` at the project root (Hexo merges this with the theme's own `_config.yml`). Key aspects:

- **Renderer**: Pug + Stylus + EJS
- **Custom scripts** in theme's `scripts/`: tags (chart, gallery, tabs), filters (lazyload, post_image, default cover assignment)
- Navigation, aside, and page layouts are all configured from `_config.solitude-kai.yml`

### Sitemap

Custom sitemap with tiered priority and image support:

| File | Purpose |
|---|---|
| `sitemap-template.xml` | Custom Nunjucks template (overrides hexo-generator-sitemap default) |
| `scripts/sitemap-enhance.js` | `before_generate` filter — marks posts updated within 90 days as `_sitemap_recent` |

Priority tiers: homepage (1.0 daily) > sticky/recent posts (0.8 weekly) > static pages (0.6 monthly) > archived posts (0.5 monthly) > tags/categories (0.3 monthly). Posts with `cover` frontmatter get `<image:image>` tags with `<image:title>` and `<image:caption>` (from excerpt).

Config in `_config.yml`:
```yaml
sitemap:
  path: sitemap.xml
  template: sitemap-template.xml
  rel: true
  tags: true
  categories: true
```

### Shared Build Config

`build.config.json` provides domain/name/email/author/output-dir defaults consumed by both `generate-error-pages.mjs` and `generate-compressed.mjs`. CLI arguments override config values.

### Key Hexo Plugins

| Plugin | Purpose |
|---|---|
| `hexo-abbrlink` | CRC32-based permalinks (`p/:abbrlink.html`) |
| `hexo-generator-sitemap` | Sitemap generation (custom template) |
| `hexo-generator-feed` | Atom feed at `/atom.xml` |
| `hexo-filter-nofollow` | `rel="nofollow"` on external links (excludes own domains) |
| `hexo-wordcount` | Post word count and reading time |
| `hexo-renderer-markdown-it` | Markdown with KaTeX math, merge-cells, post_asset images |
| `@renbaoshuo/markdown-it-katex` | KaTeX math rendering |

## Image Conventions

All images in `source/` follow strict naming (enforced by project memory):

- **Lowercase, kebab-case, ASCII only** — no Chinese, no spaces, no uppercase, no special chars
- **Extension**: Prefer `.avif`, lowercase (`.avif` not `.AVIF`)
- **Post images**: `source/_posts/[post-name]/photo-NN.avif` (two-digit numbering)
- **Directory-specific naming**: see `[[image-naming-convention]]` in project memory for full rules per subdirectory (`img/gear/`, `img/icons/`, `img/showcase/`, etc.)

## Scaffold

`scaffolds/draft.md` is the template for new posts with frontmatter fields: `title`, `categories`, `tags`, `cover`, `excerpt`. New posts start as drafts (`default_layout: draft` in config).
